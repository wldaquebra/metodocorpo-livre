Workflow — Preview + Publicação

Como visualizar alterações antes de publicar no site oficial

Ferramentas utilizadas

GitHub Pages — site oficial, o que o público vê. Só atualiza quando você manda subir no main.
Netlify — usado apenas para prévia. Gera uma URL temporária para você visualizar qualquer alteração antes de publicar. Não publica no site oficial (está bloqueado).

Flow completo — passo a passo

1
Peça a alteração
Fale o que quer mudar. Ex: "mude o preço para R$ 47,50".
2
Um branch de preview é criado
A alteração vai para um branch separado (ex: preview-preco-4750), não para o main. O site oficial continua intacto.
3
Netlify gera a URL de preview
Em ~30 segundos aparece um novo deploy na aba Deploys do Netlify. Clique nele para abrir a URL e visualizar como ficou.
4A
Gostou? → Fale "sobe no main"
A alteração vai para o main e o GitHub Pages atualiza o site em ~1 minuto.
4B
Não gostou? → Fale "descarta"
O branch é removido e o site oficial continua sem nenhuma alteração.

Configuração do Netlify (feita uma vez por projeto)

1
Conecte o repositório GitHub ao Netlify.
2
Vá em Project configuration → Continuous deployment → Branches and deploy contexts → Configure
Mude Branch deploys para All.
3
Na página Deploys, clique em "Lock to stop auto publishing".
Isso impede que o Netlify publique o site no main — ele vira só ferramenta de preview.

Resumo rápido

Quer testar → pede o branch → vê no Netlify → gostou → sobe no main → GitHub Pages atualiza.
Nunca precisa fazer nada manualmente além de aprovar ou descartar a alteração.
